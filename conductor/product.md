# 产品指南 (Product Guide)

## 1. 初始构想 (Initial Concept)
`dz-vehicle-v2` 是一个专为企业内部设计的**无人值守车辆资产借还系统**。它通过集成 OTP（动态口令）验证和全自动化的工作流，解决了传统钥匙和加油卡管理中对人工依赖强、记录不透明、预警不及时等痛点。系统旨在提供一个安全、高效且完全本地化的管理方案。

## 2. 目标用户 (Target Users)
- **借用人员 (Staff)**：需要随时借用车辆和加油卡的普通员工。
- **管理员 (Admins)**：负责资产配置、用户管理以及监控车辆合规状态（年检、保险等）的管理人员。

## 3. 核心目标 (Core Goals)
- **全自动化 (Automation)**：实现 24/7 全天候无需人工干预的自助借还。
- **安全性 (Security)**：通过 OTP 验证和白名单机制，确保每一笔借还都可追溯到人。
- **合规管理 (Compliance)**：主动追踪车辆资产的合规性，通过 UI 高亮和预警通知降低逾期风险。

## 4. 重点功能 (Key Features)
- **数据管理 (Data Mgmt)**：当前的重点是加强 CLI 批量导入工具的健壮性，包括更严格的错误检查和数据校验，确保基础数据的准确性。
- **无人值守流程**：极简的 UI 操作，支持一键借用“车辆+加油卡”组合。
- **实时看板**：带 Excel 风格过滤的公共借还历史面板，数据透明可见。
- **预警系统**：自动识别并提醒即将到期的保养、年检和保险。

## 5. 核心限制与要求 (Constraints)
- **纯中文 UI**：为了降低内部员工的使用门槛，所有界面必须仅使用简体中文。
- **容器化部署**：系统必须支持通过 Docker 进行一键式生产环境部署，简化运维难度。
